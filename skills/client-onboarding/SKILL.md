# Client Onboarding

Automatisiertes Onboarding für neue Mediapol-Kundenprojekte.

## Onboarding-Checklist

### Phase 1: Projekt-Setup (Tag 1)

\`\`\`
☐ 1. Jira-Projekt erstellen
    - Projektname: [Kundenname] - [Projektname]
    - Key: [3-4 Buchstaben]
    - Template: Kanban oder Scrum
    - Board konfigurieren (Columns: To Do, In Progress, Review, Done)

☐ 2. Confluence-Space erstellen
    - Space Name: [Kunde] - [Projekt]
    - Startseite mit Projektübersicht
    - Unterseiten: Requirements, Meeting Notes, Technical Docs

☐ 3. Mission Control Board erstellen
    - Board: "[Kunde] - [Projekt]" im Kundenprojekte-Space
    - Agenten zuweisen (Forge, Anvil, Pixel, Compass)

☐ 4. Git Repository anlegen (falls nötig)
    - GitHub/GitLab Repo erstellen
    - Standard-Struktur (README, .gitignore, CI/CD)
    - Zugriffsrechte konfigurieren
\`\`\`

### Phase 2: Kommunikation (Tag 1-2)

\`\`\`
☐ 5. Welcome-Email an Kunden
    - Projektübersicht und Timeline
    - Ansprechpartner und Kommunikationskanäle
    - Nächste Schritte

☐ 6. Kickoff-Meeting vorbereiten
    - Agenda erstellen
    - Teilnehmer einladen
    - Präsentation mit Projektscope, Timeline, Milestones

☐ 7. Discord-Kanal erstellen (intern)
    - #[kunde]-[projekt]
    - Relevante Agenten hinzufügen
\`\`\`

### Phase 3: Projektstart (Woche 1)

\`\`\`
☐ 8. Initiale Tasks erstellen
    - User Stories / Epics in Jira
    - Erste Sprint-Planung
    - Technische Architektur-Tasks

☐ 9. Monitoring einrichten
    - Projekt in Sales-Pipeline als "Gewonnen" markieren
    - Reporting-Zyklus festlegen (wöchentlich/monatlich)
\`\`\`
